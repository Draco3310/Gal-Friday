# Contributing to Gal-Friday Crypto Trading Bot

## Introduction

Thank you for considering contributing to Gal-Friday. This document outlines the guidelines and best practices for contributing to this project.

## Code of Conduct

By participating in this project, you are expected to uphold our Code of Conduct. Please report unacceptable behavior to byoung3310@gmail.com .

## How to Contribute

### Reporting Bugs

1. **Check Existing Issues** - Look through the existing issues to see if someone else has reported the same issue. If an issue already exists, feel free to add additional information or reactions.
2. **Create a New Issue** - If your issue does not exist, create a new issue and provide as much detail as possible to help us understand the problem.

### Proposing Enhancements

1. **Check Existing Issues and Discussions** - Look for existing discussions about the proposed feature.
2. **Create a New Issue or Discussion** - Clearly describe your feature and why it would be beneficial to the project.

### Code Contributions

1. **Fork the Repository** - Fork the Gal-Friday repository to your GitHub account.
2. **Clone the Fork** - Clone your fork locally.
3. **Create a New Branch** - Create a new branch for your feature or fix.
4. **Make Changes** - Make your changes and commit them to your branch.
5. **Run Tests** - Ensure that your changes pass all the tests.
6. **Submit a Pull Request** - Submit a pull request from your fork to the Gal-Friday repository.

## Development Environment

- Python 3.x
- Libraries: NumPy, Pandas, Joblib, Sentry, Backtrader, TA-Lib, QuantConnect/Lean

## Testing

Run `test_modules.py` to execute the test suite.
